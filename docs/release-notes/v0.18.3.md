## 🇩🇪 AeroACARS v0.18.3 — Absicherung LAN-Fernbedienung

Kleines Nachbesserungs-Release direkt im Anschluss an v0.18.2.

**Fix: fehlender Schutz gegen doppelten Touchdown-Sampler bei aktivierter LAN-Fernbedienung.** Wer die LAN-Fernbedienung (Settings → Fernbedienung) nutzt, um AeroACARS von einem Tablet oder zweiten PC im selben Netzwerk mitzusteuern, konnte in einer seltenen Konstellation zwei parallele Touchdown-Erfassungen für denselben Flug bekommen — z. B. wenn sowohl die Haupt-App als auch die Fernbedienungs-Oberfläche unabhängig voneinander einen unterbrochenen Flug fortsetzen. Die anderen beiden betroffenen Hintergrundprozesse (Positions-Übertragung, Live-Tracking) hatten bereits einen Schutz dagegen; der Touchdown-Sampler war die einzige Ausnahme. Jetzt ebenfalls abgesichert.

Kein Wire-Format-Wechsel, kein Handlungsbedarf für aeroacars-live.

## 🇬🇧 AeroACARS v0.18.3 — LAN remote-control hardening

Small follow-up release right after v0.18.2.

**Fix: missing protection against a duplicate touchdown sampler when the LAN remote control is enabled.** Pilots using the LAN remote control (Settings → Remote Control) to operate AeroACARS from a tablet or a second PC on the same network could, in a rare scenario, end up with two parallel touchdown captures for the same flight — e.g. if both the main app and the remote-control interface independently resumed an interrupted flight. The other two background processes this can affect (position upload, live tracking) already had a safeguard against this; the touchdown sampler was the only exception. Now protected as well.

No wire-format change, no aeroacars-live deployment needed.
