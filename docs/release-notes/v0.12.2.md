## 🇬🇧 English

**v0.12.2 — study-level X-Plane aircraft detected properly, Discord diagnostics.**

### What's new

#### ✈️ X-Plane: the real fix for study-level add-ons (CL650)

Study-level X-Plane add-ons such as the Hot Start Challenger 650 run cockpit and system functions through their **own** DataRefs and don't drive the standard `sim/...` ones. That's why AeroACARS never "saw" the CL650 flaps (the GSG225 report). v0.12.1 caught this fail-soft — v0.12.2 ships the **real** fix:

A new **aircraft DataRef profile system** detects a known study-level aircraft (via the Web API aircraft title **or** an RREF probe on a signature DataRef) and then subscribes to that add-on's own DataRefs. First profile: the **Hot Start Challenger 650** — flaps, battery master, beacon and taxi light are now read correctly. The CL650 flap lever (0/20/30/45) is verified against the manufacturer documentation, so the LANDING CONFIG check and approach-stability scoring work fully for the CL650 again.

If the pilot swaps aircraft at runtime, the adapter automatically falls back to the standard DataRefs. Aircraft without a profile are unchanged.

#### 🎮 Discord: push diagnostics in the settings panel

The Discord panel (Settings → Discord) now shows **"Last Discord push: X s ago"** plus the last error message. During an active flight this time must stay fresh every few seconds — if it stalls, the live presence isn't getting through. This makes a Discord display problem diagnosable right in the client instead of guessing.

### What pilots need to do

**Nothing** — the auto-updater fetches v0.12.2 automatically. Settings + landing history are preserved. CL650 pilots on X-Plane don't need to configure anything — the profile kicks in automatically.
