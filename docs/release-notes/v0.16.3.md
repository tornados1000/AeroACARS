# v0.16.3 — Aerosoft A340-600 (MSFS): Triebwerke, Fuel Flow & Autopilot nativ

🇩🇪 **Deutsch**

## Verbesserung (Aerosoft A346 Pro, MSFS)

- Der Aerosoft A340-600 (ToLiss-Port) bedient einige Standard-Werte des Sims
  nicht — AeroACARS musste Triebwerksstatus und Spritverbrauch bisher über
  Umwege schätzen (N1-Fallback, Verbrauch aus der Tankmenge abgeleitet).
- **Durch eine Analyse des Flugzeug-Pakets haben wir die echten Quellen
  gefunden** und lesen jetzt nativ:
  - **Triebwerksstatus** über die vom A346 bediente Variante
    (`GENERAL ENG COMBUSTION EX1`) — Engine-Start/-Stop wird sofort und
    pro Triebwerk korrekt erkannt.
  - **Fuel Flow** über `TURB ENG CORRECTED FF` — echter Verbrauch statt
    Ableitung, genauere Fuel-Statistiken im PIREP.
  - **Autopilot-Status** über die A346-eigenen Cockpit-Signale
    (AP1/AP2/APPR/LOC) — das Activity-Log zeigt AP-Ein/Aus jetzt korrekt.
- **Andere Flugzeuge sind nicht betroffen:** die neuen Quellen greifen nur,
  wenn das Flugzeug sie liefert (bzw. beim A346-Profil); alle bisherigen
  Erkennungen und Fallbacks bleiben unverändert.

## Hinweis

Reines Client-Update — aktualisiert sich automatisch.

🇬🇧 **English**

## Improvement (Aerosoft A346 Pro, MSFS)

- The Aerosoft A340-600 (a ToLiss port) doesn't drive some of the sim's
  standard values — AeroACARS had to estimate engine state and fuel burn via
  workarounds (N1 fallback, burn derived from tank quantity).
- **An analysis of the aircraft package revealed the real sources**, now read
  natively:
  - **Engine state** via the variant the A346 actually serves
    (`GENERAL ENG COMBUSTION EX1`) — start/stop detected instantly, per engine.
  - **Fuel flow** via `TURB ENG CORRECTED FF` — real burn instead of a
    derivation; more accurate fuel stats in the PIREP.
  - **Autopilot state** via the A346's own cockpit signals (AP1/AP2/APPR/LOC) —
    the activity log now shows AP on/off correctly.
- **Other aircraft are unaffected:** the new sources only apply when the
  aircraft serves them (or under the A346 profile); all existing detections
  and fallbacks stay unchanged.

## Note

Pure client update — auto-updates.
