## 🇩🇪 AeroACARS v0.18.1 — Aircraft-Scan, Falcon 50 & Zibo 737 (X-Plane)

Größeres Feature-Update. Fasst mehrere Wochen Arbeit an einem neuen Werkzeug und zwei neuen Premium-Flugzeugprofilen zusammen.

**Neu: Aircraft-Scan.** Über die Einstellungen (Settings → Aircraft-Scan) kannst du dein aktuell geladenes Flugzeug-Add-on direkt einreichen — ohne den kompletten (oft mehrere GB großen) Community-/Add-on-Ordner manuell hochzuladen. AeroACARS wählt selbst nur die relevanten Konfigurations- und Code-Dateien aus (bei MSFS: cfg/json/xml/js/wasm, bei X-Plane: acf/lua/xpl — keine Texturen, Modelle oder Sounds), verpackt sie und schickt sie an unseren Server. Damit können wir viel schneller neue, saubere Telemetrie-Profile für weitere Flugzeuge bauen. Die Einreichung ist komplett freiwillig; du siehst deine eigenen Einreichungen und deren Status jederzeit unter live.kant.ovh/aircraft und kannst deine Daten dort selbst löschen (der anonymisierte technische Bericht bleibt, damit die Analyse nicht verloren geht). Details zur Datenverarbeitung stehen in der Datenschutzerklärung.

**Neu: Contrail Falcon 50 Premium-Profil.** Vollständige Erkennung des Trijets inklusive FMA (Flight-Mode-Annunciator: aktive/armierte Lateral- und Vertikal-Modi), Autothrottle-Status und Master-Warning/Feuerwarnung je Triebwerk.

**Neu: Zibo/Laminar 737-800 Premium-Profil (X-Plane).** Sauberere AP-Modus-Anzeige (Heading, Altitude, LNAV, Approach) über die dokumentierten `laminar/B738`-Annunciator-Datarefs — funktioniert sowohl für das Standard-Laminar-737 als auch für das beliebte Zibo-Mod, da beide dieselben Kern-Datarefs treiben.

**Fix: Auto-File hing beim Falcon 50.** Der Falcon 50 meldet nach dem Abstellen der Triebwerke weiterhin "Triebwerk läuft" — dadurch wartete Auto-File endlos. Für dieses konkrete, bestätigte Flugzeug erkennt AeroACARS jetzt zusätzlich: voll gestanden am Zielflughafen für 5 Minuten, Parkbremse gesetzt → geflogen. Dieser Zusatz-Fallback gilt bewusst nur für Flugzeuge mit nachgewiesenem Triebwerkszähler-Fehler (aktuell der Falcon 50) und ändert nichts am Verhalten aller anderen Flugzeuge — ein normaler, längerer Rollhalt (z. B. Warten auf weitere Rollfreigabe an einem großen Flughafen, auch mit gesetzter Parkbremse) wird dadurch bei keinem anderen Muster fälschlich als Ankunft gewertet.

**Fix: Startrolle-Erkennung nach Startabbruch.** Ein abgebrochener Start (zurück zum Rollhalt) konnte die Phasen-Erkennung in einem falschen Zustand hängen lassen; behoben.

**Fix: Flugzeugtyp-Erkennung gehärtet.** Rund 42 % der intern geloggten Flugzeugtyp-Werte waren Datenmüll (z. B. "NONE", leere Strings) statt eines echten ICAO-Codes — die Ableitung ist jetzt deutlich robuster.

**Für Entwickler:** neue GitHub-Actions-CI baut und testet bei jedem Push (nicht nur bei Release-Tags) — Regressionen fallen jetzt sofort auf, nicht erst beim nächsten Release.

## 🇬🇧 AeroACARS v0.18.1 — Aircraft-Scan, Falcon 50 & Zibo 737 (X-Plane)

A larger feature update, bundling several weeks of work on a new tool and two new premium aircraft profiles.

**New: Aircraft-Scan.** From Settings → Aircraft-Scan you can submit your currently loaded aircraft add-on directly — no need to manually upload your entire (often multi-GB) Community/add-on folder. AeroACARS automatically selects only the relevant configuration and code files (MSFS: cfg/json/xml/js/wasm; X-Plane: acf/lua/xpl — no textures, models, or sounds), packages them, and sends them to our server. This lets us build clean new telemetry profiles for more aircraft much faster. Submission is entirely voluntary; you can see your own submissions and their status anytime at live.kant.ovh/aircraft, and delete your data there yourself (the anonymized technical report stays so the analysis isn't lost). Details on data processing are in the privacy policy.

**New: Contrail Falcon 50 premium profile.** Full detection of the trijet including FMA (Flight Mode Annunciator: active/armed lateral and vertical modes), autothrottle status, and master-warning/fire-warning per engine.

**New: Zibo/Laminar 737-800 premium profile (X-Plane).** Cleaner AP mode display (Heading, Altitude, LNAV, Approach) via the documented `laminar/B738` annunciator datarefs — works for both the stock Laminar 737 and the popular Zibo mod, since both drive the same core datarefs.

**Fix: auto-file got stuck on the Falcon 50.** The Falcon 50 keeps reporting "engine running" after shutdown, causing auto-file to wait indefinitely. For this specific, confirmed aircraft, AeroACARS now additionally detects: fully stopped at the destination airport for 5 minutes with the parking brake set → arrived. This extra fallback deliberately applies only to aircraft with a confirmed engine-counter fault (currently the Falcon 50) and changes nothing for every other aircraft — a normal, longer taxi hold (e.g. waiting for further taxi clearance at a busy airport, even with the parking brake set) is never mistaken for an arrival on any other type.

**Fix: takeoff-roll detection after a rejected takeoff.** An aborted takeoff (back to the holding point) could leave phase detection in a stuck state; fixed.

**Fix: hardened aircraft-type detection.** Around 42% of internally logged aircraft-type values were garbage (e.g. "NONE", empty strings) instead of a real ICAO code — the derivation is now considerably more robust.

**For developers:** a new GitHub Actions CI now builds and tests on every push (not just release tags) — regressions surface immediately instead of at the next release.
