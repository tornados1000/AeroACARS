# AeroACARS v0.8.5 — Landing-Analyse-Bundle: Banner ↔ PIREP konsistent + UX-Verbesserungen für VA-Admins

## 🇩🇪 Deutsch

### Was war kaputt

Der Stable-Approach-Gate-Banner im Live-Cockpit prüfte bisher mit großzügigen Schwellen:

- Gear ≥ 95 % (Backend: ≥ 99 %)
- Flaps ≥ 20 % (Backend: ≥ 70 %)

Resultat: Live-Banner sagte „Config OK", aber die PIREP-Analyse danach markierte denselben Flug als „Landing-Config NICHT GESETZT". Piloten haben den Widerspruch in Discord-Reports gemeldet — zu Recht.

Die Backend-Werte 0.99 / 0.70 sind FAA-AC-120-71B-konform. Der Banner war einfach historisch zu lax.

### Was sich für Piloten ändert

Der Banner im Cockpit nutzt jetzt dieselben Schwellen wie die PIREP-Analyse:

- **Gear ≥ 99 %** (statt 95 %)
- **Flaps ≥ 70 %** (statt 20 %)

Beim Flug **wird der Banner früher Alarm geben** als bisher — wenn du z. B. nur Flaps 30 % bei 1000 ft AGL hast, fliegst du nicht stable nach FAA. Vorher hat AeroACARS das toleriert, im PIREP danach aber bemängelt.

**Bonus:** Wenn der Banner triggert, sagt er jetzt konkret was fehlt:

> Vorher: `Config nicht gesetzt`
> Jetzt: `Config: Gear 87 %/Flaps 60 %`

So weißt du sofort wo du nachregeln musst.

## 🇬🇧 English

**v0.8.5 fixes a consistency bug between the live cockpit banner and the PIREP analysis, and ships six UX improvements to the VA-admin evaluation (aeroacars-live webapp) at the same time.**

### What was broken

The Stable-Approach-Gate banner in the live cockpit was using lenient thresholds:

- Gear ≥ 95 % (backend: ≥ 99 %)
- Flaps ≥ 20 % (backend: ≥ 70 %)

Result: the live banner said "Config OK" but the PIREP analysis afterwards flagged the same flight as "Landing-Config NOT SET". Pilots reported the contradiction in Discord — rightly so.

The backend values 0.99 / 0.70 are FAA AC 120-71B compliant. The banner was simply historically too lax.

### What changes for pilots

The cockpit banner now uses the same thresholds as the PIREP analysis:

- **Gear ≥ 99 %** (instead of 95 %)
- **Flaps ≥ 70 %** (instead of 20 %)

During flight **the banner will warn you earlier** than before — if you only have Flaps 30 % at 1000 ft AGL, you're not stable per FAA. Before, AeroACARS tolerated it, but flagged it in the PIREP afterwards.

**Bonus:** when the banner triggers, it now tells you exactly what's missing:

> Before: `Config not set`
> Now: `Config: Gear 87 %/Flaps 60 %`

So you immediately know what to fix.

### In parallel on the VA-admin webapp (aeroacars-live, live.kant.ovh)

Six UX improvements in the PIREP detail that are already live and affect all existing PIREPs retroactively — no re-compute needed:

| Item | What | Visible where |
|---|---|---|
| **Coaching tips sorted** | Critical hints (stall, wing-strike, sink) on top instead of scattered, with colour code 🔴🟠⚪ | "What you can improve next time" |
| **Aim + TCH combo tip** | If TCH too low AND touchdown past-aim → clear "duck-under" hint | Approach Stability tips |
| **Vref source trust** | Vref-Deviation tile now shows in the label whether PMDG FMC (precise), FBW (precise) or ICAO default (±5 kt approximate) | Landing Quality card |
| **Data source line** | Header shows simulator + Vref source as "📡 Data source: MSFS · Vref via PMDG FMC (precise)" | Approach Stability card |
| **Sub-score tooltips** | Hover over score pills (sink rate, G-force, ...) now shows rationale + coaching tip | PIREP feed list |
| **Banner wording** | "NOT SET" instead of "MISSING", "Stable Approach Gate" tooltip without aviation jargon | Approach Stability config chip |
