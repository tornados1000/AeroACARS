# v0.16.9 — X-Plane: Pushback-Hänger behoben, OFP-Treue wieder bewertbar

🇩🇪 **Deutsch**

## Bugfix (X-Plane, alle Addons mit eigenem Flugmodell — z. B. FF/LevelUp 767)

- **Phase hing in „Pushback"** durch den kompletten Taxi-out und Startlauf,
  weil das Vorwärts-Signal dieser Addons dauerhaft 0 meldet (Live-Flug BCS 8).
  Das Signal wird jetzt aus den simulator-eigenen Kinematik-Daten berechnet
  (Welt-Geschwindigkeit + Heading) — die sind bei **jedem** Addon lebendig.
  Taxi-out, Takeoff-Roll und Takeoff werden wieder normal erkannt.
- **„OFP-Treue: nicht bewertet (Tank-Daten fehlen)"** auf solchen Flügen
  behoben — Folge des Pushback-Hängers: die Takeoff-Phase wurde übersprungen
  und der Takeoff-Fuel nie festgehalten. Zusätzlich gibt es jetzt ein
  Sicherheitsnetz: selbst wenn die Takeoff-Phase aus irgendeinem Grund
  übersprungen wird, werden Takeoff-Zeit, -Fuel und -Gewicht nachgelatcht —
  der Trip-Burn-Vergleich gegen das OFP bleibt bewertbar.
- Plausibilitäts-Wächter: widerspricht das Bewegungs-Signal dem Groundspeed,
  gilt es als „nicht vorhanden" (dokumentierter Fallback) statt die
  Phasen-Logik einzufrieren — schützt vor künftigen Addon-Eigenheiten.
- Ehrlicherer Hinweistext: statt „Tank-Daten fehlen" steht jetzt
  „Takeoff-Daten nicht erfasst (Takeoff-Phase übersprungen)".

## Hinweis

- MSFS ist nicht betroffen (natives körperfestes Signal, unverändert).
- Landing-Score-Berechnung unverändert (Replay-Tests byte-identisch).

---

🇬🇧 **English**

## Bugfix (X-Plane, all addons with an external flight model — e.g. FF/LevelUp 767)

- **Phase stuck in "Pushback"** through the entire taxi-out and takeoff roll,
  because these addons report a permanently-zero forward-motion signal
  (live flight BCS 8). The signal is now derived from the simulator's own
  kinematic state (world velocity + heading), which is alive on **every**
  addon. Taxi-out, takeoff roll and takeoff are detected normally again.
- **"OFP fidelity: not scored (fuel data missing)"** on such flights fixed —
  a consequence of the pushback hang: the takeoff phase was skipped and the
  takeoff fuel never latched. There is also a new safety net: even if the
  takeoff phase is skipped for any reason, takeoff time, fuel and weight are
  back-filled — the trip-burn comparison against the OFP stays scoreable.
- Plausibility guard: if the motion signal contradicts the groundspeed it is
  treated as "not available" (documented fallback) instead of freezing the
  phase logic — protects against future addon quirks.
- More honest hint text: instead of "fuel data missing" it now reads
  "takeoff data not captured (takeoff phase skipped)".

## Notes

- MSFS is unaffected (native body-frame signal, unchanged).
- Landing-score computation unchanged (replay tests byte-identical).
