# v0.15.23 — Kurze/niedrige VFR-Flüge (Busch): Landung wird erfasst & akzeptiert

🇩🇪 **Deutsch**

## Bugfix (Touchdown-Erfassung auf kurzen/niedrigen Flügen)

- **Kurze, niedrige VFR-Flüge (z. B. Buschtour-Hops von Piste zu Piste)
  wurden teils zur manuellen Prüfung gehalten — „kein Touchdown".** Ursache:
  ein Flug, der landet, **bevor** er über 500 ft AGL steigt, blieb in der
  Phase „Start" hängen. Die Touchdown-Erfassung war auf die Flugphasen
  beschränkt (Start war ausgenommen) → der reale Aufsetzpunkt wurde **nicht
  erfasst** und auch **nicht an den Server gemeldet** → der Flug ging zur
  Prüfung.
- **Jetzt wird der Aufsetzpunkt auch in der Start-Phase erfasst**, sobald die
  Maschine tatsächlich geflogen ist (≥ 50 ft AGL) — also genau bei den kurzen
  Pisten-zu-Pisten-Hops. Zusätzlich wird der erfasste Touchdown jetzt **immer
  zuverlässig an den Server gemeldet** (vorher konnte er gescort, aber bei
  diesen Flügen nicht gesendet werden) → keine falschen „kein Touchdown"-Holds
  mehr.
- **Der Phantom-Schutz bleibt vollständig:** Boden-/Roll-/Taxi-Phasen bleiben
  ausgeschlossen (ein Fahrwerks-Wackler beim Rollen wird nie als Landung
  gewertet), der erste echte Aufsetz-Edge gewinnt, und die bewährte
  Touchdown-Validierung läuft unverändert davor. **Normale Flüge sind
  unverändert.**

## Hinweis

Reines Client-Update — keine Server-/Plugin-Änderung nötig, der Client
aktualisiert sich automatisch.

---

🇬🇧 **English**

## Bugfix (touchdown capture on short/low flights)

- **Short, low VFR flights (e.g. bush-tour strip-to-strip hops) were
  sometimes held for manual review — "no touchdown".** Cause: a flight that
  lands **before** climbing through 500 ft AGL stayed in the "Takeoff" phase.
  Touchdown capture was limited to the flight phases (Takeoff was excluded) →
  the real touchdown was **not captured** and **not reported to the server** →
  the flight went to review.
- **The touchdown is now captured in the Takeoff phase too**, once the
  aircraft has genuinely flown (≥ 50 ft AGL) — exactly the short strip-to-strip
  hop case. The captured touchdown is now also **always reliably reported to
  the server** (previously it could be scored but, on these flights, never
  sent) → no more false "no touchdown" holds.
- **Phantom protection is fully preserved:** ground/rollout/taxi phases stay
  excluded (a taxi gear-bump is never treated as a landing), the first genuine
  touchdown edge wins, and the proven touchdown validation still runs in front.
  **Normal flights are unchanged.**

## Note

Pure client update — no server / plugin change needed; the client auto-updates.
