## 🇩🇪 Deutsch

**v0.7.16 — Fenix A32x als Opt-in-Beta. Standardmäßig deaktiviert, kein Risiko für Nicht-Fenix-Piloten. Wer Fenix fliegt, kann das neue Profil in den Settings selbst einschalten.**

### Was sich ändert (für alle Piloten)

**Nichts** — solange du den Schalter nicht antippst.

v0.7.16 ist ein normales Stable-Release. Wenn der neue **Fenix A32x Beta**-Schalter aus bleibt (Default), läuft AeroACARS bit-identisch zu v0.7.15.

### Was neu ist (für Fenix-Piloten als Opt-in)

Im Settings-Panel gibt es jetzt eine neue Sektion **Beta-Features** mit dem Schalter **Fenix A32x Beta**. Wer ihn einschaltet, bekommt:

#### A319 / A320 / A321 Variant-Erkennung
- vorher: alle drei Fenix-Varianten liefen als „Fenix A320" im PIREP
- jetzt: korrekte Variant-Erkennung — A319 / A320 / A321 erscheint korrekt in Activity-Log und PIREP
- Detection per Title + ICAO-Fallback (auch Repaints ohne Variant-Suffix werden sauber erkannt)

#### Zusätzliche Fenix-LVAR-Mappings
Nur aktiv, wenn der Schalter **Settings → Beta-Features → Fenix A32x Beta** angeklickt ist:

| Schalter im Cockpit | LVAR | wirkt auf |
|---|---|---|
| Landing Lights L/R Selektor | `S_OH_EXT_LT_LANDING_L`/`_R` | `light_landing` |
| Nose-Light (off/taxi/T.O.) | `S_OH_EXT_LT_NOSE` | `light_taxi` |
| Wing-Inspection | `S_OH_EXT_LT_WING` | `light_wing` (neu für Fenix) |
| Runway-Turnoff | `S_OH_EXT_LT_RWY_TURNOFF` | nur read-only QS |
| Landing Both Composite | `S_OH_EXT_LT_LANDING_BOTH` | nur Verifikation gegen L/R |
| Flaps-Lever-Detent | `S_FC_FLAPS` | nur read-only QS |

LVAR-Namen wurden gegen den **echten Fenix-Install** verifiziert
(`SimObjects\Airplanes\FNX_32X\model\FNX32X_Interior.xml`).

### Wichtiger Hinweis

> Die Fenix-Funktion ist eine **Beta-Funktion**. Sie ist standardmäßig deaktiviert und muss in den Einstellungen manuell aktiviert werden. Ohne Aktivierung ändert sich für Fenix- und Nicht-Fenix-Piloten nichts.

### Was Fenix-Tester tun

```
1. AeroACARS v0.7.16 installieren (normaler Auto-Updater)
2. Settings → Beta-Features → "Fenix A32x Beta" anschalten
3. Flug mit Fenix A319 / A320 / A321 starten
4. QS-Matrix aus dem Beta-Guide abarbeiten (Cold & Dark, Taxi, T/O, Cruise, App, LDG)
5. Feedback in Discord posten
```

**Im MSFS Community-Ordner ist nichts zu tun.** Kein FSUIPC, kein WASM-Modul, keine SDK-Datei. AeroACARS liest die LVARs direkt per SimConnect, Fenix exposed sie selbst.

### Was die Beta absichtlich NICHT macht

- Keine Schalter setzen, keine Knobs drehen, keine Fenix-Systeme steuern
- Keine FMC- / MCDU-Daten
- Keine Flightplan-Manipulation
- Keine FSUIPC-Abhängigkeit
- Kein Eingriff in PIREP-Pflichtfelder

### Sicherheit / Daten

- Wenn Flag aus: bit-identisches Verhalten zu v0.7.15 — sowohl für Fenix- als auch für Nicht-Fenix-Piloten
- Wenn Flag an und LVAR fehlt: leise Fallback auf Standard-MSFS-SimVar, kein Crash
- Pilot kann den Schalter jederzeit wieder ausschalten — wirkt sofort, kein Sim-Neustart

## 🇬🇧 English

**v0.7.16 — Fenix A32x as an opt-in beta. Off by default, zero risk for non-Fenix pilots. Fenix pilots can enable the new profile in Settings themselves.**

### What changes (for everyone)

**Nothing** — as long as you don't toggle the switch.

v0.7.16 is a regular stable release. With the new **Fenix A32x Beta** toggle left off (the default), AeroACARS behaves bit-identically to v0.7.15.

### What's new (opt-in for Fenix pilots)

The Settings panel now has a new **Beta features** section with a **Fenix A32x Beta** toggle. Enabling it adds:

#### A319 / A320 / A321 variant detection
- before: all three Fenix variants showed up as "Fenix A320" in the PIREP
- now: proper variant detection — A319 / A320 / A321 is shown correctly in the activity log and PIREP
- detection via title + ICAO fallback (repaints without a variant suffix are still recognised)

#### Additional Fenix LVAR mappings
Only active when **Settings → Beta features → Fenix A32x Beta** is enabled:

| Cockpit switch | LVAR | maps to |
|---|---|---|
| Landing lights L/R selector | `S_OH_EXT_LT_LANDING_L`/`_R` | `light_landing` |
| Nose light (off/taxi/T.O.) | `S_OH_EXT_LT_NOSE` | `light_taxi` |
| Wing inspection | `S_OH_EXT_LT_WING` | `light_wing` (new for Fenix) |
| Runway turnoff | `S_OH_EXT_LT_RWY_TURNOFF` | read-only QS only |
| Landing both composite | `S_OH_EXT_LT_LANDING_BOTH` | verification against L/R |
| Flaps lever detent | `S_FC_FLAPS` | read-only QS only |

LVAR names verified against the **live Fenix install**
(`SimObjects\Airplanes\FNX_32X\model\FNX32X_Interior.xml`).

### Important notice

> The Fenix feature is a **beta feature**. It is disabled by default and must be enabled manually in Settings. Without enabling it, nothing changes for Fenix or non-Fenix pilots.

### What Fenix testers do

```
1. Install AeroACARS v0.7.16 (normal auto-updater)
2. Settings → Beta features → toggle "Fenix A32x Beta" on
3. Start a flight with the Fenix A319 / A320 / A321
4. Work through the QS matrix from the beta guide (cold & dark, taxi, T/O, cruise, app, LDG)
5. Post feedback in Discord
```

**Nothing to do in the MSFS Community folder.** No FSUIPC, no WASM module, no SDK file. AeroACARS reads the LVARs directly via SimConnect; Fenix exposes them itself.

### What this beta deliberately does NOT do

- No switch flipping, no knob turning, no control of Fenix systems
- No FMC / MCDU data
- No flight-plan manipulation
- No FSUIPC dependency
- No interference with PIREP mandatory fields

### Safety / data

- Flag off: bit-identical behaviour to v0.7.15 — for both Fenix and non-Fenix pilots
- Flag on and LVAR missing: silent fallback to standard MSFS SimVar, no crash
- The toggle can be turned off again at any time — takes effect immediately, no sim restart
