# v0.15.12 — Aircraft-Aliase greifen jetzt auch nach „Resume"

🇩🇪 **Deutsch**

## Bugfix (Aircraft-Type-Aliase / Mismatch nach Resume)

- **Die „⚠ Aircraft-Mismatch nach Resume"-Warnung ignorierte die Aircraft-
  Type-Aliase.** Beim Flugstart und Auto-Start wurden Aliase (sowohl die
  eingebaute Tabelle als auch die im Recorder-Admin gepflegten VPS-Aliase)
  korrekt berücksichtigt — der Check **nach einer Pause/Resume** machte
  dagegen einen stumpfen Direktvergleich von Sim-ICAO und Bid-ICAO und las
  **gar keine Aliase**. Folge: Falsch-Warnungen nach Resume, obwohl ein
  passender Alias gepflegt war (z. B. X-Plane meldet `PHENOM30`, Bid erwartet
  `E55P`).
- **Jetzt nutzt der Resume-Check dieselbe alias-bewusste Logik wie der
  Flugstart-Gate** (`clean_atc_model`-Normalisierung + Title-Fallback +
  eingebaute & VPS-Aliase). Damit gilt konsistent: **einmal im Recorder-Admin
  einen Alias pflegen → überall ruhig**, auch nach Resume. Kein Pilot-Client-
  Release mehr nötig, um *neue* Alias-Fälle abzudecken.
- Die Meldung bleibt eine reine **Warnung** (informieren statt blockieren) —
  ein echter Flugzeug-Wechsel (z. B. Bid `B737` / 737-700, geladen `B738` /
  737-800) wird weiterhin korrekt angezeigt.

## Hinweis

Reines Client-Update. Baut auf v0.15.11 auf.

---

🇬🇧 **English**

## Bugfix (aircraft-type aliases / mismatch after resume)

- **The "⚠ Aircraft mismatch after resume" warning ignored the aircraft-type
  aliases.** Flight start and auto-start correctly honoured aliases (both the
  built-in table and the VPS aliases maintained in the recorder admin), but the
  check **after a pause/resume** did a blunt direct comparison of sim ICAO vs
  bid ICAO and read **no aliases at all**. Result: false warnings after resume
  even when a matching alias existed (e.g. X-Plane reports `PHENOM30`, bid
  expects `E55P`).
- **The resume check now uses the same alias-aware logic as the flight-start
  gate** (`clean_atc_model` normalisation + title fallback + built-in & VPS
  aliases). So it is now consistent: **add an alias once in the recorder admin →
  quiet everywhere**, resume included. No pilot-client release needed to cover
  *new* alias cases.
- The message stays a pure **warning** (inform, don't block) — a genuine
  aircraft change (e.g. bid `B737` / 737-700, loaded `B738` / 737-800) is still
  surfaced correctly.

## Note

Pure client update. Builds on v0.15.11.
