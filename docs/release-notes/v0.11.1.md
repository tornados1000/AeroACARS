## 🇩🇪 Deutsch

**v0.11.1 — Hotfix: Pilot-Client-Version war im PIREP-Payload nicht enthalten.**

### Was war kaputt

v0.11.0 brachte die Pilot-Client-Versions-Pill in den VPS-Webapp-Report (`📱 AeroACARS vX.Y.Z`) — sichtbar in der Reports-Übersicht und im Landing-Analysis-Detail. Die UI-Render-Logik war korrekt, aber das `client_version`-Feld kam nie im Touchdown-Payload an: der Code in `aeroacars-mqtt::FlightMeta` setzte das Feld nur in der **Connect-Message** (Sitzungs-Start), NICHT im `TouchdownPayload` und auch nicht im `PirepPayload`. Resultat: die Pill blieb für alle Flüge leer.

### Was diese Version fixt

- `TouchdownPayload` enthält jetzt `client_version: Some(env!("CARGO_PKG_VERSION"))`
- `PirepPayload` ebenfalls
- Beide werden bei jedem Touchdown / PIREP-Submit gesetzt → Webapp kann die Pill jetzt aus jeder Touchdown-DB-Row direkt anzeigen

### Was Piloten tun müssen

**Nichts** — der Auto-Updater holt v0.11.1 automatisch. Settings + Landing-History bleiben erhalten.

Hinweis: die Pill erscheint nur für Flüge die du **nach** dem Update auf v0.11.1 einreichst. Pre-v0.11.1-PIREPs (auch die aus v0.11.0) haben das Feld nicht im Payload — sie zeigen weiterhin keine Pill.

## 🇬🇧 English

**v0.11.1 — Hotfix: pilot client version was missing from PIREP payload.**

### What was broken

v0.11.0 added the pilot-client version pill to the VPS webapp report (`📱 AeroACARS vX.Y.Z`) — visible in the reports overview and in the landing-analysis detail. The UI render logic was correct, but the `client_version` field never arrived in the touchdown payload: the code in `aeroacars-mqtt::FlightMeta` only set the field in the **connect message** (session start), NOT in `TouchdownPayload` and not in `PirepPayload` either. Result: the pill stayed empty for every flight.

### What this version fixes

- `TouchdownPayload` now includes `client_version: Some(env!("CARGO_PKG_VERSION"))`
- `PirepPayload` too
- Both are set on every touchdown / PIREP submit → webapp can now show the pill directly from every touchdown DB row

### What pilots need to do

**Nothing** — the auto-updater fetches v0.11.1 automatically. Settings + landing history are preserved.

Note: the pill only appears for flights you submit **after** updating to v0.11.1. Pre-v0.11.1 PIREPs (including those from v0.11.0) don't have the field in the payload — they will continue to show no pill.
