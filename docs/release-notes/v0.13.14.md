# v0.13.14 — Landungs-Report geht nicht mehr verloren (MQTT retained)

🇩🇪 **Deutsch**

## Was ist neu

- **Fix: verlorene Landungs-Reports / fehlende Score-Aufschlüsselung.** Am
  Flugende sendet AeroACARS zwei MQTT-Nachrichten — `touchdown` und `pirep`
  (mit der vollständigen Score-Aufschlüsselung). Bisher wurden beide
  **nicht-retained** gesendet: war der Live-Server (Recorder) in genau dem
  Moment kurz offline (Neustart, Mosquitto-Reload, Netz-Glitch), ging die
  Nachricht **für immer verloren**. Folge: der Touchdown war im Live-Dashboard
  da, aber **ohne verknüpften PIREP** — die „Score-Aufschlüsselung" blieb leer
  und die angezeigte Note konnte falsch sein (vorläufiger Touchdown-Wert statt
  finalem Flug-Score).
- **Lösung:** `touchdown` und `pirep` werden jetzt **retained** publiziert
  (wie `position`/`phase`/`status` schon immer). Mosquitto speichert die letzte
  Nachricht pro Pilot — ein Recorder, der kurz nach dem Flugende wieder
  verbindet, holt sie sofort nach. Kein Report geht mehr verloren.
- **Sicher:** Eine erneut zugestellte Nachricht erzeugt **kein** Duplikat —
  der Recorder ist idempotent (PIREP eindeutig per `pirep_id`, Touchdown per
  Zeitfenster-Deduplizierung). Die nächste Landung desselben Piloten
  überschreibt den gespeicherten Wert.

## Für VA-Owner

- **Keine Wire-Format-Änderung.** `aeroacars-live` muss **nicht** mit-deployed
  werden — der Recorder verarbeitet retained-Nachrichten bereits korrekt.
  Reines Client-Update.

---

🇬🇧 **English**

## What's new

- **Fix: lost landing reports / missing score breakdown.** At flight end
  AeroACARS publishes two MQTT messages — `touchdown` and `pirep` (the latter
  carrying the full score breakdown). Both were previously sent
  **non-retained**: if the live server (recorder) was briefly offline at that
  exact moment (restart, mosquitto reload, network blip), the message was
  **lost for good**. Result: the touchdown showed up in the live dashboard but
  **with no linked PIREP** — the "score breakdown" stayed empty and the shown
  grade could be wrong (a provisional touchdown value instead of the final
  flight score).
- **Solution:** `touchdown` and `pirep` are now published **retained** (like
  `position`/`phase`/`status` always have been). Mosquitto keeps the last
  message per pilot, so a recorder reconnecting shortly after the flight ends
  picks it up immediately. No report is lost.
- **Safe:** A re-delivered message does **not** create a duplicate — the
  recorder is idempotent (PIREP unique by `pirep_id`, touchdown by time-window
  dedup). The pilot's next landing overwrites the retained value.

## For VA owners

- **No wire-format change.** `aeroacars-live` does **not** need to be
  re-deployed — the recorder already handles retained messages. Pure client
  update.
