# v0.13.16 — Auto-Start blockiert nicht mehr das manuelle Starten · Aircraft-Aliase wirken sofort

🇩🇪 **Deutsch**

## Was ist neu

- **Fix: Auto-Start verhinderte das manuelle „Trotzdem starten".** Stand das
  Flugzeug am Abflughafen und passte der Flugzeug-Typ nicht zum Bid (z. B.
  Wetlease, oder nur ein fehlender Typ-Alias wie Phenom 300), feuerte der
  Auto-Start-Watcher alle 3 Sekunden einen Startversuch, der jeweils am
  Aircraft-Mismatch scheiterte. Jeder dieser Versuche belegte kurz den
  Start-Lock — genau dann schnappte er dem Piloten den Klick auf „Trotzdem
  starten" weg, mit der roten Meldung *„another flight start or adopt is
  already in progress"*.
- **Lösung:** Der Auto-Start prüft den Flugzeug-Typ jetzt **vorab** (mit
  derselben Logik wie der manuelle Start, inklusive Aircraft-Type-Aliases) und
  feuert nur noch, wenn er auch durchgehen würde. Passt der Typ nicht, hält er
  sich zurück und stört das manuelle Starten nicht mehr. Sobald du das passende
  Flugzeug lädst (oder ein Alias ergänzt wird), greift der Auto-Start beim
  nächsten Check **automatisch** — kein Aus/An-Toggeln, kein Neustart nötig.
  Bei einem echten Mismatch erscheint ein klarer Hinweis im ACARS-Log.
- **Fix: Aircraft-Type-Aliase wirken sofort.** Ein vom VA frisch eingetragener
  Typ-Alias (z. B. „E55P ⇒ PHENOM 300") wurde bisher erst beim übernächsten
  Flug bzw. nach einem Neustart wirksam. Jetzt wird die Alias-Liste **beim
  Flugstart frisch geladen** (parallel, ohne spürbare Wartezeit) — der Alias
  greift damit schon beim nächsten Startversuch.

## Für VA-Owner

- **Keine Wire-Format-Änderung.** `aeroacars-live` muss **nicht** mit-deployed
  werden — reines Client-Update.
- **Tipp:** Wenn ein Flugzeug zwar passt, aber Bid-ICAO und Sim-Name
  unterschiedlich heißen (z. B. `E55P` vs `PHENOM 300E`), trag im Recorder-
  Admin unter **Navdata → Aircraft-Aliases** ein Mapping ein (ICAO `E55P`,
  Alias `PHENOM 300`). Es gilt sofort und für alle Airlines/Registrierungen
  desselben Typs — kein Client-Release nötig.

Dank an **Michael K** für den Befund (DWW306 EDTM→EHAM, Phenom 300 / D-CFHZ).

🇬🇧 **English**

## What's new

- **Fix: auto-start blocked manual "Start anyway".** When the aircraft was
  parked at the departure airport and its type didn't match the bid (e.g. a
  wetlease, or just a missing type alias like the Phenom 300), the auto-start
  watcher fired a start attempt every 3 seconds, each failing on the aircraft
  mismatch. Every attempt briefly held the start lock — exactly when it stole
  the pilot's "Start anyway" click, producing the red *"another flight start or
  adopt is already in progress"* message.
- **Fix:** Auto-start now **pre-checks the aircraft type** (using the same logic
  as a manual start, including aircraft-type aliases) and only fires when it
  would actually succeed. On a mismatch it stays out of the way and no longer
  interferes with starting manually. As soon as you load the matching aircraft
  (or an alias is added), auto-start picks it up **automatically** on the next
  check — no toggling off/on, no restart. A clear hint is shown in the ACARS log
  on a real mismatch.
- **Fix: aircraft-type aliases take effect immediately.** A type alias the VA
  just added (e.g. "E55P ⇒ PHENOM 300") previously only applied from the
  flight-after-next or after a restart. The alias list is now **refreshed at
  flight start** (in parallel, with no noticeable delay) — so the alias applies
  on the very next start attempt.

## For VA owners

- **No wire-format change.** `aeroacars-live` does **not** need to be
  re-deployed — pure client update.
- **Tip:** If an aircraft matches but the bid ICAO and the sim name differ
  (e.g. `E55P` vs `PHENOM 300E`), add a mapping in the recorder admin under
  **Navdata → Aircraft Aliases** (ICAO `E55P`, alias `PHENOM 300`). It takes
  effect immediately and for every airline/registration of the same type — no
  client release needed.

Thanks to **Michael K** for the report (DWW306 EDTM→EHAM, Phenom 300 / D-CFHZ).
