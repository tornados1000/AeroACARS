## 🇩🇪 Deutsch

**v0.8.1 — Hotfix: MSFS-Vendor-Tag-Prefix `$$:` im ATC-MODEL-String wird jetzt automatisch entfernt. Erkennt damit Aircraft die manche Addon-Pakete (Flysimware Citation X, einige Carenado-Modelle) mit einem `$$:`-Präfix melden.**

### Was sich ändert

#### Aircraft-Mismatch-Check toleranter gegen Vendor-Tags

**Problem:** Beim Flugstart-Check vergleicht AeroACARS den Bid-ICAO (z. B. `C750` für Citation X) gegen das was MSFS in der `ATC MODEL` SimVar liefert. Bei einigen Addon-Paketen kommt da ein Hersteller-Tag mit, z. B. `$$:C750` statt nur `C750`. Folge: Pre-Flight-Banner „Flugzeug-Typ passt nicht — bid wants C750, sim has $$:C750" — obwohl es dasselbe Aircraft ist.

**Fix (`clean_atc_model`):** Wenn der ATC-MODEL-String mit einem kurzen Prefix beginnt der aus 1–4 Sonderzeichen + `:` besteht (z. B. `$$:`, `##:`), wird der Prefix gestrippt bevor der Match-Check läuft. `$$:C750` → `C750` → match mit Bid `C750`.

**Schutz gegen Kollateral-Schäden:**
- Buchstaben-Präfixe wie `TT:` (text-token) werden weiter abgewiesen (war pre-v0.8.1-Verhalten).
- ATCCOM-Strings (`ATCCOM.AC_MODEL_C750.0.TEXT`) parsen weiterhin über den AC_MODEL-Substring-Pfad korrekt.

### Anchor-Case

GSG / Sven M, 13.05.2026: Flug GSG200 mit Cessna Citation X (HDJT-Bid… moment, C750-Bid). MSFS-Addon meldete `$$:C750`. Pre-Flight blockierte. Mit v0.8.1: durchgewunken.

### Was Piloten tun müssen

Nichts — Auto-Updater rollt v0.8.1 aus. Beim nächsten Flugstart: $$-Präfix-Aircraft werden automatisch akzeptiert ohne Mismatch-Banner.

### Alternative ohne Pilot-Client-Update

VA-Admins können auch via Webapp-Navdata-Tab → „Aircraft-Type-Aliases" einen Eintrag pflegen (`ICAO C750 → Alias $$:C750`). Beide Pfade funktionieren parallel — die Hardcoded-Lösung in v0.8.1 entlastet aber den Admin von dieser Pflege-Aufgabe für ähnliche Fälle.

## 🇬🇧 English

**v0.8.1 — Hotfix: MSFS vendor-tag prefix `$$:` in the ATC MODEL string is now stripped automatically. Recognises aircraft that some addon packs (Flysimware Citation X, certain Carenado models) report with a `$$:` prefix.**

### What changes

#### Aircraft-mismatch check tolerant of vendor tags

**Problem:** At flight start AeroACARS compares the bid ICAO (e.g. `C750` for the Citation X) against what MSFS reports in the `ATC MODEL` SimVar. Some addon packs prepend a manufacturer tag, e.g. `$$:C750` instead of just `C750`. The pre-flight banner then said "Aircraft mismatch — bid wants C750, sim has $$:C750" — even though it's the same aircraft.

**Fix (`clean_atc_model`):** When the ATC MODEL string starts with a short prefix of 1–4 non-alphanumeric characters + `:` (e.g. `$$:`, `##:`), the prefix is stripped before the match check runs. `$$:C750` → `C750` → matches bid `C750`.

**Collateral-damage guards:**
- Letter-prefixes like `TT:` (text-token) keep being rejected (pre-v0.8.1 behaviour).
- ATCCOM strings (`ATCCOM.AC_MODEL_C750.0.TEXT`) keep parsing via the AC_MODEL substring path.

### Anchor case

GSG / Sven M, 2026-05-13: flight GSG200 with the Cessna Citation X. The MSFS addon reported `$$:C750`. Pre-flight blocked. With v0.8.1: cleared.

### What pilots need to do

Nothing — the auto-updater rolls v0.8.1 out. From the next flight start, `$$:`-prefixed aircraft are accepted automatically without the mismatch banner.

### Alternative without pilot-client update

VA admins can also curate this via the webapp Navdata tab → "Aircraft-Type Aliases" (`ICAO C750 → Alias $$:C750`). Both paths work in parallel — the hardcoded solution in v0.8.1 relieves the admin of that maintenance for similar future cases.
