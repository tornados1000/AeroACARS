**v0.13.0 — Sim-Glitches werden ab sofort erkannt. PIREPs aus kaputten Flügen (Sim-Crash, Sim-Pause, Slew-Mode an, Teleport) landen automatisch in der Review-Queue beim VA-Owner statt einfach durchgewunken zu werden.**

### Was sich für Dich als Pilot ändert

**Bei einem normalen, sauberen Flug:** Nichts. PIREP geht durch wie immer.

**Bei einem komischen Flug** (Sim crasht, Sim-Pause während Cruise, Slew-Mode aus Versehen an, lange MQTT-Verbindungsabbrüche): während des Flugs kommt ein farbiger Banner oben im Cockpit:

- 🟧 **Oranger Banner** "Datenanomalie" — z. B. ein Telemetrie-Lücke. Kannst Du wegklicken, beeinflusst PIREP nur leicht (geht meistens noch als trusted durch).
- 🟥 **Roter Banner** "Data-Integrity-Problem" — z. B. SIM_STATE_RESET nach Sim-Pause. **Lässt sich nicht wegklicken.** Heißt: Dein PIREP wird wahrscheinlich nicht automatisch akzeptiert, sondern landet beim VA-Owner zur manuellen Prüfung.

Der Banner ist **keine Drohung**, sondern eine Vorwarnung: "He, hier ist was Komisches passiert — der VA-Owner wird das sehen."

### Was sich für Dich als VA-Owner ändert (Thomas)

Im phpVMS-Admin gibt's einen neuen Sidebar-Eintrag **🛡 AeroACARS Integrity**:

- **Settings + Verbindungs-Test** zum Recorder
- **Review-Queue**: Liste aller geheldeten PIREPs mit Flags + Reasons + welcher Pilot + welche Route
- **Sicherheits-Status**: zeigt grün wenn alle Ranks korrekt auf `auto_approve_acars=false` stehen (das Modul setzt das automatisch beim ersten Start + täglich + bei jedem PIREP)

Saubere PIREPs gehen wie immer automatisch durch ACCEPTED. Nur die mit Anomalien bleiben PENDING — Du entscheidest manuell ob Du sie akzeptierst, mit Warnung akzeptierst, ablehnst oder als false positive verwirfst.

### 🛑 Sim-Crash-Schutz (Stream F mit Pilot-getriggertem Re-Check)

**Das ist das eigentliche WZZ2271-Fix.** Wenn der AeroACARS-Client erkennt dass die aktuelle Sim-Position stark von der gespeicherten Flug-Position abweicht (= Sim wurde nach Crash neu geladen, Slew-Mode versetzt das Flugzeug, oder Position-Drift > 50 nm), zeigt der Banner jetzt einen **Drei-Optionen-Dialog ohne Countdown**:

1. 🟢 **Hauptknopf (grün, primary): "Position jetzt prüfen + fortsetzen"** — der Pilot bringt sich **erst manuell** im Sim wieder zur gespeicherten Position (Slew-Mode oder Reload), drückt **dann** diesen Button. Der Client misst die aktuelle Drift:
   - **Wenn < 5 nm:** Flug läuft **sauber weiter, kein untrusted-Stempel.** Zeit-Gap bis ~10 min ist OK solange die Position passt.
   - **Wenn ≥ 5 nm:** Banner zeigt aktuelle Drift, Pilot kann nochmal positionieren + nochmal prüfen
2. 🔴 **Danger-Button (rot): "Flug verwerfen + neu starten"** — alter Flug wird als cancelled auf phpVMS markiert, Du wählst neuen Bid
3. 🟧 **Versteckter Erweitert-Knopf: "Trotzdem fortsetzen (untrusted)"** — Notfall-Override, PIREP geht in die Review-Queue beim VA-Owner

**Kein 30s-Countdown mehr, kein silent-auto-resume — Du musst aktiv klicken.**

#### 📊 Live-Vergleichs-Tabelle im Banner

Das rote Banner zeigt jetzt direkt ein **3-Spalten-Grid (Gespeichert | Sim jetzt | Δ)** für **Position, Altitude, Aircraft, Fuel, ZFW und Total Weight**:

```
                GESPEICHERT       SIM JETZT        Δ

```

Die Sim-Spalte wird **live** (1×/s) aus dem aktuellen Sim-Snapshot aktualisiert. Sobald Du im Sim slewst oder Fuel im EFB nachstellst, siehst Du die Δ-Werte schrumpfen und von rot ⚠ auf grün ✓ kippen. Damit weißt Du genau **wann Du auf "Position prüfen + fortsetzen" klicken darfst** — kein Trial-and-Error mehr.

Toleranzen für grün ✓: Position < 5 nm, Altitude < 500 ft, Fuel < 500 kg, Weight < 1000 kg, Aircraft = exakter ICAO-Match.

Workflow nach Sim-Crash:
1. AeroACARS startet → erkennt aktiven Flug → Drift-Check → suspect=true
2. Banner zeigt 3 Optionen
3. Pilot startet Sim wieder, positioniert sich zur letzten gemerkten Position (z. B. via Slew-Mode + Coordinates aus der Flug-Anzeige)
4. Pilot drückt "Position jetzt prüfen + fortsetzen"
5. **Wenn passt:** "Position OK — Drift 1.2 nm. Flug läuft jetzt als sauber weiter." → Resume ohne Strafe
6. **Wenn nicht passt:** "Drift 47 nm — bitte näher positionieren und nochmal prüfen." → Pilot probiert nochmal

**v0.13.0 — Sim glitches are now detected. PIREPs from broken flights (sim crash, sim pause, slew mode on, teleport) automatically go into the VA owner's review queue instead of being silently accepted.**

### What changes for you as a pilot

**On a normal, clean flight:** Nothing. PIREP goes through as always.

**On a weird flight** (sim crashes, sim pause during cruise, accidentally slew-mode on, long MQTT disconnects): during the flight a colored banner appears at the top of the cockpit:

- 🟧 **Orange banner** "Data anomaly" — e.g. a telemetry gap. You can dismiss it; PIREP is only mildly affected (usually still trusted).
- 🟥 **Red banner** "Data integrity problem" — e.g. SIM_STATE_RESET after sim pause. **Cannot be dismissed.** Means: your PIREP will probably not be auto-accepted but goes to the VA owner for manual review.

The banner is **not a threat**, it's a heads-up: "Hey, something weird happened — the VA owner will see this."

### What changes for you as VA owner (Thomas)

In phpVMS admin there's a new sidebar entry **🛡 AeroACARS Integrity**:

- **Settings + connection test** to the recorder
- **Review queue**: list of all held PIREPs with flags + reasons + which pilot + which route
- **Safety status**: shows green when all ranks are correctly set to `auto_approve_acars=false` (the module sets this automatically on first boot + daily + per PIREP)

Clean PIREPs pass through ACCEPTED as always. Only those with anomalies stay PENDING — you decide manually whether to accept, accept-with-warning, reject, or dismiss as false positive.

### 🛑 Sim crash protection (Stream F with pilot-triggered re-check)

**This is the actual WZZ2271 fix.** When the AeroACARS client detects that the current sim position differs strongly from the saved flight position (= sim reloaded after crash, slew mode moved the aircraft, or position drift > 50 nm), the banner now shows a **three-option dialog without countdown**:

1. 🟢 **Primary button (green): "Check position now + resume"** — the pilot **first manually** repositions in the sim back to the saved position (slew mode or reload), **then** presses this button. Client measures current drift:
   - **If < 5 nm:** flight resumes **cleanly, no untrusted stamp.** Time gap up to ~10 min is OK as long as position matches.
   - **If ≥ 5 nm:** banner shows current drift, pilot can reposition + check again
2. 🔴 **Danger button (red): "Discard flight + start over"** — old flight marked as cancelled on phpVMS, you pick a new bid
3. 🟧 **Hidden Advanced button: "Continue anyway (untrusted)"** — emergency override, PIREP goes to VA owner's review queue

**No 30s countdown anymore, no silent auto-resume — you must click actively.**

#### 📊 Live comparison table in the banner

The red banner now shows a **3-column grid (Saved | Sim now | Δ)** for **Position, Altitude, Aircraft, Fuel, ZFW and Total Weight**:
