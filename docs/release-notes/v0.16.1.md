# v0.16.1 — LAN-Fernbedienung: weiße Seite behoben

🇩🇪 **Deutsch**

## Bugfix (LAN-Fernbedienung)

- **Beim Öffnen der LAN-Adresse auf dem Tablet/zweiten Rechner blieb die Seite
  weiß bzw. zeigte „Seite nicht gefunden".** Ursache: der Server suchte die
  Oberfläche als Dateien auf der Platte — in der installierten App ist das
  Frontend aber **fest eingebaut** (nicht als lose Dateien), darum wurde nichts
  gefunden.
- **Jetzt liefert der Server die eingebaute Oberfläche korrekt aus** (über
  Tauris eingebettete Assets). Karte, Cockpit, Log, alle Buttons laden auf dem
  Tablet — getestet (echte Seite + Assets werden ausgeliefert, die PIN-/API-
  Absicherung bleibt intakt).
- Klein nebenbei: ein Power-User-Schalter zum automatischen Starten des Servers
  beim App-Start (Umgebungsvariable `AEROACARS_LAN_AUTOSTART=<port>`).

## Hinweis

Reines Client-Update — aktualisiert sich automatisch. Behebt das v0.16.0-
Problem; die LAN-Fernbedienung funktioniert jetzt wie vorgesehen.

🇬🇧 **English**

## Bugfix (LAN remote control)

- **Opening the LAN address on the tablet/second computer showed a blank page
  or "page not found".** Cause: the server looked for the UI as files on disk,
  but in the installed app the frontend is **embedded** (not loose files), so
  nothing was found.
- **The server now serves the embedded UI correctly** (via Tauri's embedded
  assets). Map, cockpit, log, all buttons load on the tablet — verified at
  runtime (the real page + assets are served; the PIN/API protection stays
  intact).
- Small extra: a power-user switch to auto-start the server on app launch
  (env var `AEROACARS_LAN_AUTOSTART=<port>`).

## Note

Pure client update — auto-updates. Fixes the v0.16.0 issue; LAN remote control
now works as intended.
