# v0.15.26 — Activity-Log überlebt jetzt einen Neustart mitten im Flug

🇩🇪 **Deutsch**

## Bugfix (ACARS-Activity-Log)

- **Wenn man den Client mitten im Flug beendet, er abstürzt oder ein Update
  macht, war das ACARS-Activity-Log nach dem Neustart leer** (auf macOS und
  Linux). Die komplette Flugchronik (Phasenwechsel, Gear, Lights, Engines,
  Fuel/Weight, Pausen …) war scheinbar weg.
- **Jetzt wird das Log beim Start wiederhergestellt** — die gesamte bisherige
  Chronik des laufenden Flugs ist wieder da, oben drauf der „App gestartet"-
  Marker. So kann man den Client mittendrin neu starten/aktualisieren, ohne den
  Verlauf zu verlieren.
- Hintergrund: das Log wurde immer schon **gespeichert** (bei jedem Eintrag),
  aber der **Wiederherstellungs-Pfad war Windows-only** — auf Mac/Linux wurde
  die falsche Stelle gesucht und nichts gefunden. Jetzt nutzt das Laden
  **denselben plattformübergreifenden Pfad wie das Speichern**.

## Hinweis

Reines Client-Update — aktualisiert sich automatisch.

---

🇬🇧 **English**

## Bugfix (ACARS activity log)

- **When you closed / crashed / updated the client mid-flight, the ACARS
  activity log was empty after restart** (on macOS and Linux). The whole flight
  history (phase changes, gear, lights, engines, fuel/weight, pauses …) appeared
  to be gone.
- **The log is now restored on startup** — the full history of the in-progress
  flight is back, with the "app started" marker on top. So you can restart or
  update the client mid-flight without losing the history.
- Background: the log was always **saved** (on every entry), but the **restore
  path was Windows-only** — on Mac/Linux it looked in the wrong place and found
  nothing. Loading now uses the **same cross-platform path as saving**.

## Note

Pure client update — auto-updates.
