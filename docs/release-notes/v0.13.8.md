## 🇬🇧 English

**v0.13.8 — phpVMS PIREP performance. You noticed big flights take long to open on the phpVMS site — root cause was a too-fat position payload. Fix: send slimmer.**

### What you get

- **phpVMS PIREP detail page opens noticeably faster** for long flights. For a 3.5h flight:
  - Before: ~860 positions × ~600 bytes = **~500 KB** PIREP size
  - After: ~430 positions × ~150 bytes = **~65 KB** = **-87%**
- **VPS recorder + live map unchanged** — they still receive every telemetry field over a separate MQTT stream. Forensics, replay, landing analysis: all as before.

### What specifically changed

- **Cruise cadence: 30s → 60s.** At 480 kt GS that's ~8 nm between points — plenty for the route trail on the map.
- **Per-position `log` field disabled.** Previously every position carried a JSON blob with lights/COM/NAV/AP modes/state/env/ATC — 99% of that duplicated between cruise samples (wind, QNH, OAT, lights stayed constant). phpVMS stored all of it in the position row, which bloated DB joins on PIREP detail render.

### What stays the same

- **Touchdown capture** at full fidelity (5s during Landing phase)
- **Boarding/Pushback** fast (4s) — no phase skipping
- **Approach** precise (8s) — clean ILS track
- **Taxi** 10s — slow movement
- **VPS** still sees **every** snapshot via MQTT with full telemetry

Nothing. Auto-updater rolls out v0.13.8.
