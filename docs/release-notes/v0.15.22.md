# v0.15.22 — X-Plane: korrekte Vertical-Speed-Anzeige

🇩🇪 **Deutsch**

## Bugfix (X-Plane V/S-Anzeige)

- **Die V/S-Anzeige zeigte in X-Plane einen falschen Sinkflug an.** Ein
  Pilot meldete: im **Reiseflug, völlig level**, stand die V/S-Kachel auf
  z. B. **−277 fpm** statt ~0; im Sinkflug zeigte sie zu viel (−2300 →
  −2600). Ursache: die V/S kam aus der **Welt-Koordinaten-Geschwindigkeit**
  (`local_vy`), die im Level-Flug **nicht** null liest — sie trägt einen
  geschwindigkeitsabhängigen Versatz mit (das flache Welt-Frame „fällt"
  unter die gekrümmte Erde weg).
- **Jetzt liest die Anzeige den Instrument-VVI** (`vvi_fpm_pilot`), der —
  wie im Cockpit — im Level-Flug ~0 anzeigt und im Steig-/Sinkflug korrekt
  ist. Mit gefixt: die Phasen-Logik (Durchstarten-/Sinkflug-Erkennung) und
  die Anflugstabilität, die dieselbe V/S nutzen.
- **Die Landerate / Touchdown-Erfassung ist unverändert.** Sie verwendet
  weiterhin das rohe, verzögerungsfreie Signal — der Fix führt **keinen**
  Touchdown-Lag ein (der Grund, warum v0.4.3 seinerzeit auf das rohe Signal
  umgestellt hatte, bleibt gewahrt). **MSFS ist nicht betroffen** (dessen
  V/S-Wert war schon korrekt).

## Hinweis

Reines Client-Update.

🇬🇧 **English**

## Bugfix (X-Plane V/S display)

- **The V/S readout showed a false descent in X-Plane.** A pilot reported:
  in **level cruise**, the V/S tile read e.g. **−277 fpm** instead of ~0; in
  a descent it over-read (−2300 → −2600). Cause: the V/S came from the
  **world-coordinate velocity** (`local_vy`), which does **not** read zero in
  level flight — it carries a speed-dependent offset (the flat world frame
  "falls away" beneath the curved Earth).
- **The display now reads the instrument VVI** (`vvi_fpm_pilot`), which — like
  the cockpit gauge — reads ~0 in level flight and is correct in climb/descent.
  Also fixed: the phase logic (go-around / descent detection) and approach
  stability, which used the same V/S.
- **Landing rate / touchdown capture is unchanged.** It still uses the raw,
  lag-free signal — this fix introduces **no** touchdown lag (the reason
  v0.4.3 moved to the raw signal is preserved). **MSFS is unaffected** (its
  V/S value was already correct).

Pure client update.
