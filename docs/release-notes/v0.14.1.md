# v0.14.1 — Fix: Live-Map blieb in der paketierten App schwarz

🇩🇪 **Deutsch**

## Was ist neu

- **Fix: Die Live-Map (v0.14.0) zeigte in der installierten App nur eine
  schwarze Fläche** — besonders sichtbar in der **VA-Übersicht**. Ursache war
  die Sicherheits-Richtlinie (Content-Security-Policy) der App: Sie erlaubte
  den **Render-Worker** der Karte nicht, den die Kartenbibliothek aus einer
  `blob:`-Quelle startet. Ohne diesen Worker wurde gar nichts gezeichnet.
- **Lösung:** Die CSP erlaubt jetzt `worker-src`/`child-src`/`script-src blob:`
  → die Basemap und alle Kartenebenen (Route, Track, Marker, VA-Flieger)
  werden korrekt dargestellt.

> Hinweis: In der Entwickler-Version und im Browser fiel das nicht auf, weil
> dort die strenge App-CSP nicht greift — erst der signierte Build erzwingt sie.

## Für VA-Owner

- **Reines Client-Update.** Keine Server- oder Modul-Änderung, `aeroacars-live`
  muss **nicht** mit-deployed werden.

---

🇬🇧 **English**

## What's new

- **Fix: the live map (v0.14.0) showed only a black area in the installed app**
  — most visible in the **VA overview**. The cause was the app's
  Content-Security-Policy: it didn't allow the map's **render worker**, which
  the map library spins up from a `blob:` source. Without that worker nothing
  was drawn.
- **Fix:** the CSP now allows `worker-src`/`child-src`/`script-src blob:` → the
  basemap and all map layers (route, track, markers, VA aircraft) render
  correctly.

> Note: this didn't show up in the dev build or in a browser because the strict
> app CSP isn't enforced there — only the signed build enforces it.

## For VA owners

- **Pure client update.** No server or module change; `aeroacars-live` does
  **not** need to be re-deployed.
