## 🇩🇪 Deutsch

**v0.12.3 — Landing-G FOQA-konform gemessen.**

### Was ist neu

#### 🛬 G-Kraft beim Aufsetzen — wie eine echte Flugdaten-Überwachung

Bisher hat AeroACARS die Aufsetz-G-Kraft als **rohen 50-Hz-Einzelbild-Peak** bewertet. Dieser Wert ist schärfer, als ihn ein echter Flugschreiber je aufzeichnen würde — eine an sich saubere Landung konnte dadurch unnötig hart aussehen (Befund TAP533).

Ab v0.12.3 wird die G-Kraft so gemessen, wie es echte Flugdaten-Überwachung (FOQA/FDM) macht: das G-Signal wird **leicht geglättet** (gleitender Mittelwert, ~100 ms), dann der Spitzenwert genommen. Das entfernt die überscharfen Einzelbild-Ausschläge — der bewertete Wert entspricht dem, was ein Flugschreiber loggen würde. Die Messung ist **framerate-unabhängig**, also auf jedem PC reproduzierbar.

Der gescorte Wert fließt überall konsistent: G-Force-Forensik-Card, Landing-Score, Activity-Log, PIREP-Feld „Landing G-Force" und die Live-Webapp zeigen jetzt **denselben** Wert. Der rohe 50-Hz-Peak bleibt als Forensik-Detail sichtbar — nur eben nicht mehr als bewerteter Wert.

Die Crash-/Unfall-Erkennung nutzt bewusst weiter den rohen Extremwert — dort soll der schärfste Wert gesehen werden.

### Was Piloten tun müssen

**Nichts** — der Auto-Updater holt v0.12.3 automatisch. Settings + Landing-History bleiben erhalten. Bestehende PIREPs werden weiter korrekt angezeigt.

## 🇬🇧 English

**v0.12.3 — landing G measured the FOQA way.**

### What's new

#### 🛬 Touchdown G-force — measured like real flight-data monitoring

Until now AeroACARS scored the touchdown G as a **raw 50 Hz single-frame peak**. That value is sharper than any real flight recorder would ever log — so a genuinely clean landing could look needlessly hard (the TAP533 report).

From v0.12.3 the G-force is measured the way real flight-data monitoring (FOQA/FDM) does it: the G signal is **lightly smoothed** (a moving average, ~100 ms), then the peak is taken. That removes the over-sharp single-frame spikes — the scored value matches what a flight recorder would log. The measurement is **frame-rate independent**, so it's reproducible on any PC.

The scored value flows consistently everywhere: the G-Force forensics card, the landing score, the activity log, the PIREP "Landing G-Force" field and the live webapp now all show the **same** value. The raw 50 Hz peak stays visible as forensic detail — it's just no longer the scored value.

Crash/accident detection deliberately keeps using the raw extreme — there the sharpest value is the relevant one.

### What pilots need to do

**Nothing** — the auto-updater fetches v0.12.3 automatically. Settings + landing history are preserved. Existing PIREPs still display correctly.
