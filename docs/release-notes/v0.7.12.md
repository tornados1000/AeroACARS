## 🇩🇪 Deutsch

**Bid-Card: Pax + Cargo erscheinen wieder — auch ohne phpVMS-Bid-Pointer.**

### Was war kaputt

Direkt nach v0.7.11 gemeldet: bei der CFG2228-Bid (Condor) war zwischen der Aircraft-Zeile (`FLUGZEUG CFG-A21N-PW · D-ANLA`) und dem SimBrief-Plan-Block ein leerer Bereich — die `👥 169 PAX` / `📦 X t Cargo`-Chips fehlten komplett.

### Warum

v0.7.10 Pre-Flight-SimBrief-direct holt das OFP direkt von simbrief.com — ohne den phpVMS-Bid-Pointer-Cache anzufassen. Wenn der Bid noch keinen OFP via phpVMS gebunden hat (oder das phpVMS-Subfleet keine fare-entries hat), war die `subfleet.fares`-Liste leer und die Bid-Card hat die Chips ausgeblendet.

### Fix

- SimBrief-XML-Parser extrahiert jetzt `<weights><pax_count>` + `<weights><cargo>` aus dem OFP
- Die Werte werden über das `bid_simbrief_preview` Tauri-Command ans Frontend gereicht
- Bid-Card bevorzugt Preview-Werte (wenn > 0) vor den Bid-Subfleet-Fares
- Fallback identisch zum v0.3.0-Verhalten — Pilots ohne SimBrief-Settings sehen weiterhin die Bid-Pointer-Werte

Reiner UI-Hotfix für die Bid-Liste. Keine Backend-Änderung, keine PIREP-Übertragung, keine Score-Berechnung berührt.

## 🇬🇧 English

**Bid card: pax + cargo reappear — even without phpVMS bid-pointer.**

### What broke

Reported right after v0.7.11: on the CFG2228 (Condor) bid, the area between the aircraft row (`AIRCRAFT CFG-A21N-PW · D-ANLA`) and the SimBrief plan block was empty — the `👥 169 PAX` / `📦 X t Cargo` chips were completely missing.

### Why

v0.7.10 pre-flight SimBrief direct fetches the OFP straight from simbrief.com — without touching the phpVMS bid-pointer cache. When the bid hasn't bound an OFP via phpVMS yet (or the phpVMS subfleet has no fare entries), the `subfleet.fares` list was empty and the bid card hid the chips.

### Fix

- SimBrief XML parser now extracts `<weights><pax_count>` + `<weights><cargo>` from the OFP
- Values pass through the `bid_simbrief_preview` Tauri command to the frontend
- Bid card prefers preview values (when > 0) over bid subfleet fares
- Fallback identical to v0.3.0 behaviour — pilots without SimBrief settings still see the bid-pointer values

Pure UI hotfix for the bid list. No backend change, no PIREP transmission, no score calculation touched.
