# v0.15.16 — Auto-Start: Aircraft-Aliase greifen jetzt von Anfang an

🇩🇪 **Deutsch**

## Bugfix (Auto-Start / Aircraft-Type-Aliase)

- **Auto-Start zeigte bei aliased Flugzeugen dauerhaft „Flugzeug passt nicht".**
  Beispiel: Bid erwartet `E55P`, im Sim ist `PHENOM 300E` — obwohl auf dem VPS
  ein Alias `E55P → PHENOM 300` gepflegt war. Ursache: Der Auto-Start prüft den
  Flugzeug-Typ alias-bewusst, aber die VPS-Alias-Liste wurde im Client erst
  **beim Flugstart** geladen. **Vor** dem ersten Flug war die Liste leer →
  Auto-Start kannte den Alias nicht und meldete im Minutentakt „passt nicht"
  (Henne-Ei: Auto-Start bräuchte den Alias, der kam aber erst beim Start).
- **Jetzt** lädt der Auto-Start-Watcher die VPS-Aliase **selbst nach**, sobald
  die Liste leer ist (gedrosselt, sobald die Sitzung angemeldet ist). Damit
  greift ein Alias **von Anfang an** — Auto-Start erkennt den Phenom (und jedes
  andere aliased Flugzeug) automatisch, ohne dass du erst manuell „Trotzdem
  starten" musst.

## Hinweis

Reines Client-Update. Baut auf v0.15.15 auf. Tipp: Aliase pflegst du im
Recorder-Admin (live.kant.ovh) — sie greifen ohne Client-Release.

🇬🇧 **English**

## Bugfix (auto-start / aircraft-type aliases)

- **Auto-start kept showing "aircraft doesn't match" for aliased aircraft.**
  Example: bid expects `E55P`, the sim has `PHENOM 300E` — even though a VPS
  alias `E55P → PHENOM 300` existed. Cause: auto-start checks the aircraft type
  alias-aware, but the VPS alias list was only fetched in the client **at flight
  start**. **Before** the first flight the list was empty → auto-start didn't
  know the alias and reported "doesn't match" every minute (chicken-and-egg:
  auto-start needed the alias, but the alias only loaded at start).
- **Now** the auto-start watcher **warms the VPS alias list itself** whenever
  it's empty (throttled, once the session is logged in). So an alias takes
  effect **from the start** — auto-start recognises the Phenom (and any other
  aliased aircraft) automatically, without you having to "start anyway" first.

## Note

Pure client update. Builds on v0.15.15. Tip: maintain aliases in the recorder
admin (live.kant.ovh) — they take effect without a client release.
