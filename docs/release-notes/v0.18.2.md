## 🇩🇪 AeroACARS v0.18.2 — Aircraft-Scan-Nachbesserungen

Nachbesserungs-Release direkt im Anschluss an v0.18.1, gebaut aus konkreten Pilotenbefunden.

**Fix: Einreichung über den Client schlug für JEDE Datei fehl.** Der Aircraft-Scan-Upload aus dem Client (Settings → Aircraft-Scan) lehnte jede Einreichung mit "Datei zu groß" ab, unabhängig von der tatsächlichen Dateigröße — betraf jede Einreichung seit v0.18.0, nicht nur große Pakete. Ursache: ein internes ZIP-Detail führte dazu, dass der Server jede Datei fälschlich als ~4 GB groß einstufte. Behoben — Einreichungen über den Client funktionieren jetzt wie vorgesehen. (Der Web-Uploader auf live.kant.ovh/aircraft war nicht betroffen.)

**Fix: X-Plane-Flugzeuge in Kategorie-Ordnern wurden nicht gefunden.** X-Plane legt Flugzeuge nicht flach ab, sondern gruppiert sie in Kategorie-Ordnern (z. B. "Aircraft/Extra Aircraft/Zibo B738"). Die automatische Suche und der manuelle Pfad übersahen dadurch viele X-Plane-Add-ons. Behoben — auch Flugzeuge eine Ordnerebene tiefer werden jetzt gefunden.

**Neu: Ordner direkt auswählen statt Pfad tippen.** Ein "Ordner wählen…"-Button öffnet den nativen System-Dialog für den manuellen Community-/Aircraft-Pfad.

**Fix: Auto-File konnte in seltenen Fällen doppelte Einträge ins Fluglogbuch schreiben.** Bislang gab es keinen Schutz dagegen, dass versehentlich ein zweiter AeroACARS-Prozess denselben aktiven Flug parallel verfolgt (z. B. nach einem Schlaf-/Wach-Zyklus des Rechners bei aktivierter "Minimize to Tray"-Option) — das konnte zu doppelten Phasen-/Touchdown-Zeilen im PIREP führen. Ein erneuter Start der App holt jetzt nur das bestehende Fenster in den Vordergrund, statt eine zweite Instanz zu starten.

**Fix: AGL- und Gelände-Linie im Höhenprofil (Logbuch-Detail) waren praktisch unsichtbar.** Reine Darstellungssache — die Höhendaten waren immer korrekt, nur die Zeichenreihenfolge im Diagramm ließ sie hinter der MSL-Linie verschwinden. Jetzt sind alle drei Linien klar erkennbar.

## 🇬🇧 AeroACARS v0.18.2 — Aircraft-Scan follow-up fixes

Follow-up release right after v0.18.1, built from concrete pilot findings.

**Fix: client-side submission failed for EVERY file.** The Aircraft-Scan upload from the client (Settings → Aircraft-Scan) rejected every submission with "file too large", regardless of actual file size — affected every submission since v0.18.0, not just large packages. Root cause: an internal ZIP detail made the server misread every file's size as ~4 GB. Fixed — client submissions now work as intended. (The web uploader at live.kant.ovh/aircraft was not affected.)

**Fix: X-Plane aircraft in category folders weren't found.** X-Plane doesn't store aircraft flat — it groups them into category folders (e.g. "Aircraft/Extra Aircraft/Zibo B738"). Both auto-detection and manual path entry missed many X-Plane add-ons as a result. Fixed — aircraft one folder level deeper are now found too.

**New: pick a folder directly instead of typing a path.** A "Choose folder…" button opens the native system dialog for the manual Community/Aircraft path.

**Fix: auto-file could occasionally write duplicate flight-log entries.** There was previously no safeguard against a second AeroACARS process accidentally tracking the same active flight in parallel (e.g. after the computer's sleep/wake cycle with "minimize to tray" enabled) — this could cause duplicate phase/touchdown lines in the PIREP. Relaunching the app now just brings the existing window to the front instead of starting a second instance.

**Fix: AGL and terrain lines in the altitude profile (logbook detail) were practically invisible.** A pure rendering issue — the altitude data was always correct, only the chart's draw order let it disappear behind the MSL line. All three lines are now clearly visible.
