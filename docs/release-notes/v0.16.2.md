# v0.16.2 — LAN-Fernbedienung: kein API-Key auf dem Tablet mehr

🇩🇪 **Deutsch**

## Bugfix (LAN-Fernbedienung)

- **Auf dem Tablet/zweiten Rechner musste man nach der PIN nochmal den
  phpVMS-API-Key eingeben — bei jeder Sitzung.** Das ergab keinen Sinn: der
  Sim-PC ist ja bereits eingeloggt.
- Ursache: der Login-Check, mit dem die App beim Start erkennt „ich bin
  eingeloggt", war versehentlich **nicht** über die LAN-Verbindung verfügbar.
  Dadurch hielt sich die Tablet-Ansicht für ausgeloggt und zeigte die
  Login-Seite.
- **Jetzt erbt das Tablet die eingeloggte Session vom Sim-PC automatisch** —
  **du gibst den API-Key nie wieder ein.** Einmal die PIN zum Koppeln (wird
  gemerkt, bleibt auch über App-Neustarts gültig), danach öffnet sich direkt
  die App.

## Hinweis

Reines Client-Update — aktualisiert sich automatisch.

---

🇬🇧 **English**

## Bugfix (LAN remote control)

- **On the tablet/second computer you had to re-enter the phpVMS API key after
  the PIN — every session.** That made no sense: the sim PC is already logged
  in.
- Cause: the login check the app uses at startup to recognize "I'm logged in"
  was accidentally **not** exposed over the LAN connection. So the tablet view
  thought it was logged out and showed the login page.
- **The tablet now inherits the sim PC's logged-in session automatically** —
  **you never enter the API key again.** Pair once with the PIN (remembered,
  and it stays valid across app restarts), and the app opens straight away.

## Note

Pure client update — auto-updates.
