## 🇩🇪 Deutsch

**v0.9.1 — Zwei grosse neue Features auf einmal: Dein Flug wandert ins Discord-Profil, und AeroACARS-Crashes melden sich automatisch beim VA-Owner (anonym). Beides Opt-In, jederzeit abschaltbar.**

### Was ist neu

#### 🟢 Discord Rich Presence — dein Flugstatus live im Discord-Profil

Andere VA-Mitglieder sehen dich z.B. mit `GSG3184 · EDDB → KMRH` und `CRUISE · A320 · FL360` direkt in der Discord-Mitglieder-Liste — ohne dass irgendjemand das Webapp-Dashboard oder den Pilot-Client öffnen muss. Crew sieht auf einen Blick wer gerade unterwegs ist und wo.

- **Default = aus** — du musst es bewusst einschalten unter `Einstellungen → Discord Rich Presence`.
- **Anonym-Modus**: Toggle „Callsign anonymisieren" macht aus `GSG3184` ein `GSG-Flight`. Route bleibt sichtbar, dein konkretes Callsign aber nicht.
- **Sim-spezifisches Badge**: kleines MSFS-2024 / 2020 / X-Plane-11 / 12 Icon unten-rechts am AeroACARS-Logo.
- **Alle aktuell vom AeroACARS-FSM emittierten Phasen** korrekt gemappt (17 Stück, inklusive `HOLDING` und `PIREP FILED`). Für v0.10.0 sind die drei neuen Spec-Phasen `REJECTED-TAKEOFF` (⚠), `GO-AROUND` (⚠) und `DEBOARDING` bereits vorbereitet — der RPC-Code muss bei deren Aktivierung nicht angefasst werden.
- **Test-Presence-Button**: 15s Dummy senden um zu prüfen ob Discord die App sieht — ohne dafür einen echten Flug starten zu müssen.
- **Discord nicht offen?** Kein Problem. Status zeigt „Discord nicht gefunden", sonst passiert nichts.

#### 🛡 Anonyme Fehler-Telemetrie (GlitchTip)

Wenn AeroACARS crasht oder einen Fehler abfängt, kann es das jetzt automatisch an unseren eigenen Server (`tip.kant.ovh`) melden — komplett anonym. Hilft dem VA-Team Bugs zu finden, bevor sie viele Piloten treffen.

- **Default = aus.** Beim ersten Start nach dem Update siehst du ein Banner mit klarer Ja/Nein-Frage.
- **Was wird gesendet?** Crash-Stack-Trace, Sim-Name, Aircraft-ICAO, AeroACARS-Version, OS.
- **Was wird NICHT gesendet?** Position, Route, Pilot-Identität, IP-Adresse, Passwörter, E-Mail.
- **Wohin?** Eigener Server der VA, kein Sentry-3rd-Party, kein Cloud-Dienst. DSGVO Art. 6 (1) a.
- **Toggle wirkt sofort** — kein Neustart nötig, kannst jederzeit unter `Einstellungen → Fehler-Telemetrie (anonym)` umlegen.

### Was Piloten tun müssen

Nichts. Auto-Updater rollt v0.9.1 aus, beim ersten Start kommt der Datenschutz-Banner zur Fehler-Telemetrie. Beide Features sind initial ausgeschaltet — wenn du sie willst, kurz in den Settings aktivieren.

### Für VA-Owner (Thomas)

Discord-Application-ID wird im Webapp-Admin auf `live.kant.ovh/admin/` → Settings → Discord → „Discord-Application-ID" gepflegt. Pilot-Client zieht sie zur Laufzeit nach — kein Re-Release nötig wenn du die Discord-App wechselst. GlitchTip-Dashboard für Crash-Reports unter `https://tip.kant.ovh`.

## 🇬🇧 English

**v0.9.1 — Two big new features in one release: your flight shows up in your Discord profile, and AeroACARS crashes report themselves to the VA owner automatically (anonymously). Both opt-in, can be turned off any time.**

### What's new

#### 🟢 Discord Rich Presence — your flight status live in your Discord profile

Other VA members see you e.g. as `GSG3184 · EDDB → KMRH` and `CRUISE · A320 · FL360` directly in the Discord member list — without anyone having to open the webapp dashboard or the pilot client. Crew sees at a glance who's flying and where.

- **Default = off** — you have to explicitly enable it under `Settings → Discord Rich Presence`.
- **Anonymize mode**: "Anonymize callsign" toggle turns `GSG3184` into `GSG-Flight`. Route stays visible, but your specific callsign doesn't.
- **Sim-specific badge**: small MSFS 2024 / 2020 / X-Plane 11 / 12 icon in the bottom-right corner of the AeroACARS logo.
- **All flight phases currently emitted by the AeroACARS FSM** are mapped correctly (17 in total, including `HOLDING` and `PIREP FILED`). The three new spec phases `REJECTED-TAKEOFF` (⚠), `GO-AROUND` (⚠), and `DEBOARDING` are already wired up for v0.10.0 — the RPC code won't need to change when they go live.
- **Test presence button**: send a 15 s dummy to check whether Discord sees the app — no need to start a real flight to verify.
- **Discord not running?** No problem. Status shows "Discord not found", nothing else happens.

#### 🛡 Anonymous error telemetry (GlitchTip)

When AeroACARS crashes or catches an error, it can now report it automatically to our own self-hosted server (`tip.kant.ovh`) — fully anonymous. Helps the VA team find bugs before they hit many pilots.

- **Default = off.** On first start after the update you'll see a banner with a clear Yes/No question.
- **What is sent?** Crash stack trace, sim name, aircraft ICAO, AeroACARS version, OS.
- **What is NOT sent?** Position, route, pilot identity, IP address, passwords, e-mail.
- **Where to?** VA's own server, no Sentry third-party, no cloud service. GDPR Art. 6 (1) a.
- **Toggle takes effect immediately** — no restart needed, can be flipped any time under `Settings → Error telemetry (anonymous)`.

### What pilots need to do

Nothing. Auto-updater rolls v0.9.1 out, on first start you'll see the error-telemetry privacy banner. Both features are initially off — if you want them, briefly enable them in Settings.

### For the VA owner (Thomas)

Discord application ID is maintained in the webapp admin at `live.kant.ovh/admin/` → Settings → Discord → "Discord-Application-ID". Pilot client picks it up at runtime — no re-release needed if you switch the Discord app. GlitchTip dashboard for crash reports at `https://tip.kant.ovh`.
