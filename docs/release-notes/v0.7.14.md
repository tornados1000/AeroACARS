## 🇩🇪 Deutsch

**Discord-Posts laufen jetzt zentral vom VPS — Pilot-Client postet nichts mehr.**

### Warum

v0.7.13 hatte für jeden Piloten ein Settings-Feld für die Discord-Webhook-URL eingeführt. Drei Probleme:

1. **Bei N Piloten = N Stellen wo das Token leakt.** Jeder Pilot hat die URL irgendwo lokal liegen — versehentlicher Screenshot, geteilter PC, Backup-Tool → Token öffentlich.
2. **Pilot konnte eine andere URL setzen** und damit in irgendeinen anderen Discord-Channel posten oder eine bösartige URL testen.
3. **Doppelte Posts.** Der Recorder auf live.kant.ovh hat seit Monaten seinen eigenen Discord-Integration-Tab (Settings → Discord-Webhook). Der postet schon Touchdown + PIREP. Pilot-Client postete zusätzlich → doppelt.

v0.7.14 löst alles auf einen Schlag: **Pilot-Client postet gar nichts mehr in Discord**. Der Recorder ist die einzige Quelle. VA-Owner verwaltet alles in einer einzigen Webapp-Admin-Seite.

### Was Piloten merken

- Nach Update: gar nichts. Discord-Posts erscheinen automatisch sobald der VA-Owner die URL im Webapp-Admin gesetzt hat.
- Settings-Sektion „Discord-Integration" ist weg
- Alte `discord-webhook.txt` aus v0.7.13 wird beim ersten Start auf v0.7.14 automatisch gelöscht

### Was VA-Owner einmalig macht

1. Browser auf https://live.kant.ovh/admin/ → Login → **Settings** → **Discord-Webhook**
2. **Webhook-URL** einfügen (aus dem Discord-Server)
3. Toggles prüfen: **Takeoffs**, **Touchdowns**, **PIREPs** (inkl. Divert-Embed) — alle Default-On
4. **Test**-Button → „✓ Webhook OK"
5. **Speichern**

Fertig. Keine SSH, keine Env-Variable, keine Pilot-Action.

### Was im Discord ankommt (jetzt vollständig)

- **🛫 Takeoff** — neu in v0.7.14! Runway + TOW + Fuel
- **🛬 Touchdown** — wie bisher: Airport, RWY, V/S, IAS, Score, Bounce, Color-Coding
- **📋 PIREP filed** — wie bisher: Flight, Route, Block-Zeit, Fuel, Landing-V/S
- **🔀 DIVERT filed** — neu in v0.7.14! Orange Embed, Vorher→Nachher-Route klar markiert (`EDDF → ~~MDPC~~ ➜ MDST`)

### Sicherheit

- Webhook-URL liegt **nur** in der Recorder-SQLite-DB (auf VPS, chmod-protected)
- URL geht **nie** an einen Pilot-Client
- Pilot kann URL nicht sehen, nicht setzen, nicht missbrauchen
- Rotation: VA-Owner ersetzt URL in 30 Sek im Browser-UI, fertig

## 🇬🇧 English

**Discord posts now run centrally from the VPS — pilot client posts nothing.**

### Why

v0.7.13 introduced a per-pilot settings field for the Discord webhook URL. Three problems:

1. **N pilots = N places the token leaks.** Every pilot has the URL stored locally somewhere — accidental screenshot, shared PC, backup tool → public token.
2. **Pilot could set a different URL** and post into some other Discord channel or test a malicious URL.
3. **Duplicate posts.** The recorder at live.kant.ovh has had its own Discord integration tab for months (Settings → Discord Webhook). It already posts Touchdown + PIREP. Pilot client posted on top → doubled.

v0.7.14 fixes all of this at once: **pilot client posts nothing to Discord anymore**. The recorder is the single source. The VA owner manages everything in a single Webapp admin page.

### What pilots notice

- After updating: nothing. Discord posts appear automatically as soon as the VA owner has set the URL in the webapp admin.
- The "Discord Integration" settings section is gone
- Old `discord-webhook.txt` from v0.7.13 is auto-deleted on first start under v0.7.14

### What the VA owner does once

1. Browser to https://live.kant.ovh/admin/ → Login → **Settings** → **Discord Webhook**
2. Paste the **webhook URL** (from your Discord server)
3. Toggles: **Takeoffs**, **Touchdowns**, **PIREPs** (incl. divert embed) — all default-on
4. **Test** button → "✓ Webhook OK"
5. **Save**

Done. No SSH, no env variables, no pilot action.

### What lands in Discord (now complete)

- **🛫 Takeoff** — new in v0.7.14! Runway + TOW + fuel
- **🛬 Touchdown** — as before: airport, RWY, V/S, IAS, score, bounce, colour coding
- **📋 PIREP filed** — as before: flight, route, block time, fuel, landing V/S
- **🔀 DIVERT filed** — new in v0.7.14! Orange embed, before→after route clearly marked (`EDDF → ~~MDPC~~ ➜ MDST`)

### Security

- Webhook URL lives **only** in the recorder SQLite DB (on VPS, chmod-protected)
- URL **never** travels to a pilot client
- Pilot cannot see, set, or misuse the URL
- Rotation: VA owner replaces URL in 30 sec via browser UI, done
